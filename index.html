That involves the work of APIs! So What is Web API? First, we should know what API is API stands for Application Programming Interface It's an interface that provides a set of functions allowing different softwares to communicate with each other. We can think of it as if we are in a restaurant. We order our food through the waiter, waiter goes to the kitchen and then comes back with the food we ordered. Likewise, WE (the client) sends our request to the Server (the kitchen) through the help of the waiter (in this case the API) . Now what is Web API? It is a specific type of API that operates over the web. It operates using HTTP requests to enable communication between a web server and a client (a web browser, website etc.). One type of it is REST API .REST stands for “Representational State Transfer”. It's a standardized software architecture style that is industry known and used. Web companies use it, google, facebook, netflix.. So how do you make requests using the REST API? First you will need to choose the format of the data to communicate with the server. There are 2 commonly used data formats. JSON and XML. -JSON is the most commonly used format today for its simplicity and compatibility with javascript. (show an example on the slide) -XML on the other hand is less common but still used in the industry. It was popular back in the day because it looked like html. (example on the slide) -These data formats will be used in sending and receiving data to and from the server through API. Then, you will also be provided an API key - lt's used to identify who's making the request. Which is basically like your ID. You also need endpoints, which are specific URLs where the API can be accessed to perform various tasks, like retrieving data or updating info on the server. In that way API doesn't have to search the whole server but instead just target the specific part of it that has the information you need. Then we will need to specify our request. These are done through the requests method. POST: create (When you want to add data to the server) GET : read (When you want to get a specific information or data from the server) PUT: Update (When you want to modify or change something from the server) Delete: Delete (When you want to delete a data from the server) Later in the demo we will see how this information will look like in coding. How will the Server respond? First it will respond with a status code: 200-299 - request successful 400-499 - there was an error on the clients end 500-599 - there was an error on the servers end When request is successful, response will be translated in the data format you are using (JSON or XML) Now, why choose REST API. User Logs In: Authentication verifies their identity (e.g., via OAuth2). API Issues Token: User receives an API key. User Makes Request: They include the key in their request.
